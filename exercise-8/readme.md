# 8: Feature Integration

The release window is opening soon, and your team has three finished features sitting on separate branches: `feature/login-ui`, `feature/api-endpoints`, and `docs/api-guide`. 

Your team lead wants all of these integrated into the `main` branch. Thankfully, the developers communicated well, and there shouldn't be any merge conflicts. Your goal is to cleanly merge all three branches into `main` so the QA team can test the complete build.

## Objectives

1. Checkout to the `main` branch.
2. Merge the `feature/login-ui` branch into `main`.
3. Merge the `feature/api-endpoints` branch into `main`.
4. Merge the `docs/api-guide` branch into `main`.

> **Note:** If you are having trouble with VIM. Use `:wq` to write & quit the editor or use the `--no-edit` merge flag to circumvent commit message editing.
